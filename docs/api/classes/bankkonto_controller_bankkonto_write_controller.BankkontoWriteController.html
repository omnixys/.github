put

  • put(bankkontoDTO, bankkontoId, version, res): Promise<Response<any, Record<string, any>>>

  • Ein vorhandenes Bankkonto wird asynchron aktualisiert. Bei erfolgreicher Aktualisierung wird der Statuscode 204 (No Content) gesetzt und im Header auch ETag mit der neuen Version mitgeliefert.

    Falls die Versionsnummer fehlt, wird der Statuscode 428 (Precondition required) gesetzt; und falls sie nicht korrekt ist, der Statuscode 412 (Precondition failed). Falls Constraints verletzt sind, wird der Statuscode 400 (Bad Request) gesetzt.
